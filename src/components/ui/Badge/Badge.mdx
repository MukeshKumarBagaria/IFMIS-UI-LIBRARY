# Badge

A small, pill-shaped **status indicator** — an icon plus a short label. Five
semantic variants mapped to the IFMIS brand scales. Use it for list flags,
counts, and section status (the `Accordion` header uses it for completion
state).

> All examples are plain JavaScript / JSX — your modules don't need TypeScript.

---

## Variants

```jsx
import { Badge } from "@ifmis/ui";

<Badge variant="success">Complete</Badge>
<Badge variant="danger">Danger</Badge>
<Badge variant="pending">Pending</Badge>
<Badge variant="info">Info</Badge>
<Badge variant="default">Draft</Badge>   {/* or just omit `variant` */}
```

<Canvas of={BadgeStories.AllVariants} />

Each variant ships a default icon: `success` → check, `danger` → warning,
`pending` → spinner, `info`/`default` → info.

> **`info` vs `default`:** `info` is **blue** — use it for a genuinely
> informational call-out. `default` (the fallback when you omit `variant`) is
> **grey** — use it for draft / not-started / neutral states. These used to
> be the same variant (rendered grey under the name `info`), which made every
> draft badge look blue; they're separate now so each colour means one thing.

---

## Icons are dynamic

The icon is a slot:

- **omit** `icon` → the variant default.
- **`icon={null}`** → text-only.
- **`icon={<YourGlyph />}`** → custom; it inherits the variant's icon colour
  and is auto-sized to 16px.

```jsx
import { Clock } from "@phosphor-icons/react";

<Badge variant="default" icon={null}>Draft</Badge>
<Badge variant="pending" icon={<Clock weight="fill" />}>Awaiting review</Badge>
```

## Props

| Prop        | Type                               | Default  | Notes                                  |
| ----------- | ---------------------------------- | -------- | -------------------------------------- |
| `variant`   | `success \| danger \| pending \| info \| default` | `default` | Colour + default icon.   |
| `icon`      | node \| `null`                     | default  | Override the glyph, or hide with `null`. |
| `children`  | node                               | —        | The label.                            |
| `className` | string                             | —        | Merged onto the pill.                  |

Other `<span>` attributes (`id`, `data-*`, `title`, `ref`…) are forwarded.

| `danger`  | red-100     | red-400     | red-800     | red-600       |

| `default` | grey-100    | grey-400    | grey-800    | grey-600      |

Pill: `padding 4px 12px`, `radius 16px`, `gap 6px`, label 14px / 600.

## Accessibility

The label carries the meaning, so the default icons are `aria-hidden`. If you
render a badge with **no text**, add an accessible name yourself (e.g.
`aria-label`) so the status isn't lost to screen readers.

---

## Usage best practices

- **Let the label carry the meaning.** Keep it to one or two words ("Complete",
  "3 errors") — the colour is reinforcement, not the message.
- **Match the variant to status semantics**, not to decoration: `success` for
  done, `danger` for blocking, `pending` for in-progress, `info` for a genuine
  call-out, `default` (or omit `variant`) for draft/not-started/neutral.
- **Don't use a Badge as a button.** It's a static indicator — for a clickable
  chip use `SelectionPill`.

## Responsive behaviour

Badges are **inline and `shrink-0`**, so they never squash in a flex row. In a
wrapping row of badges, add a `gap` and let them flow onto multiple lines:

```jsx
<div className="flex flex-wrap items-center gap-2">
  {flags.map((f) => (
    <Badge key={f.id} variant={f.variant}>{f.label}</Badge>
  ))}
</div>
```

For a badge that must sit tight against a long, truncating title, keep the
title in a `min-w-0 truncate` element and the badge as a sibling so the badge
stays fully visible while the title ellipsises.
