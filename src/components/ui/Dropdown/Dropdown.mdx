# Dropdown

A labelled **select**. A single trigger opens a popover list of options, and
the **same** component covers single and multiple selection plus every Figma
state — default, hover, active (open), selected, preview, and disabled.

It composes [`FormField`](?path=/docs/ui-formfield-guide--docs), so the label
(with optional icon + required marker), error/helper subtext and ARIA wiring
match every other field in the library. Under the hood it's a real
`<button role="combobox">` driving a `role="listbox"`, with full keyboard
support and `aria-activedescendant`.

> All examples are plain JavaScript / JSX — your modules don't need TypeScript.

---

## Quickstart

Only `options` is required. Pass a `label` (and an optional `labelIcon`) for the
full field, and a `placeholder` for the empty state.

```jsx
import { Dropdown } from "@ifmis/ui";
import { User } from "@phosphor-icons/react";

<Dropdown
  label="Country"
  labelIcon={<User />}
  placeholder="Select a country"
  options={[
    { value: "in", label: "India" },
    { value: "np", label: "Nepal" },
  ]}
  onValueChange={(v) => console.log(v)}
/>;
```

## Single vs multiple

Add `multiple` to allow more than one selection. The popover then **stays open**
and each option toggles. `value` / `onValueChange` switch from a `string` to a
`string[]` automatically.

```jsx
<Dropdown
  multiple
  label="Countries"
  defaultValue={["in", "bt"]}
  options={countries}
  onValueChange={(values) => console.log(values)} // string[]
/>
```

<Canvas of={DropdownStories.Multiple} />

### Item visual

Each option is a **44px row** with no fill at rest. Hovering (or arrow-keying
to) a row gives it a **light** `purple-50` fill — no border. The selection
indicator differs by mode:

- **Multiple** — every row carries a small circular check on the **right**: a
  filled purple disc when selected, an outlined circle when not.
- **Single** — no circle. The selected row keeps a persistent `purple-100`
  fill, with a small purple check as a subtle confirmation.

### Chips & "+N more"

In `multiple` mode the closed trigger renders the selection as **removable
chips**. Each chip's `×` clears just that value. Past `maxVisibleChips`
(default `2`) the remainder collapse into a `+N more` affordance; clicking it —
or anywhere on the field — opens the list.

```jsx
<Dropdown
  multiple
  label="Channels"
  placeholder="Select channels"
  maxVisibleChips={2}
  defaultValue={["ussd", "mobile", "internet", "branch", "atm"]}
  options={channels}
/>
```

## Preview selection

Set `previewSelection` to list the chosen labels in a muted grey box **below**
the closed field (the Figma "Preview items" frame). It pairs naturally with
`multiple` as a compact summary of what's selected.

```jsx
<Dropdown multiple previewSelection label="Countries" options={countries} />
```

## Searchable

Add `searchable` to put a search box at the top of the popover that filters the
options as you type. It opens focused, drives the list with `↑`/`↓` and `Enter`,
and clears itself each time the menu closes. Works in single **and** multiple
mode.

```jsx
<Dropdown searchable label="Country" options={countries} />
```

<Canvas of={DropdownStories.Searchable} />

- `onSearchChange` reports the query on every keystroke — wire it to a
  server-side search if your list is large.
- `filterOption(option, query)` overrides the default match (case-insensitive
  substring on a string `label`, falling back to `value`).
- `noResultsText` customises the empty-search message.

## Positioning (auto-flip)

The popover renders in a portal and is anchored to the field, so it's never
clipped by an ancestor's `overflow`. When the field sits near the **bottom of
the viewport**, the menu automatically opens **upward** instead of being cut
off, and caps its height to the available space (scrolling inside). Pass
`flip={false}` to always open downward.

<Canvas of={DropdownStories.FlipWhenNearBottom} />

## Controlled

Drive the value from your own state with `value` + `onValueChange`:

```jsx
const [value, setValue] = useState("in");

<Dropdown
  label="Country"
  value={value}
  onValueChange={setValue}
  options={countries}
/>;
```

## Keyboard & accessibility

- **Trigger** is a `<button role="combobox">` with `aria-haspopup="listbox"`,
  `aria-expanded`, and `aria-activedescendant` pointing at the active option.
- **List** is a `role="listbox"` (`aria-multiselectable` in multiple mode); each
  item is a `role="option"` with `aria-selected`.
- **Keys:** `↓` / `↑` move the active option (skipping disabled ones), `Home` /
  `End` jump to the ends, `Enter` / `Space` select, `Esc` closes and returns
  focus to the trigger. `↓` / `Enter` / `Space` also open a closed menu.
- Clicking outside closes the menu. The label is tied to the trigger via
  `htmlFor` / `id`, so clicking it focuses the control.
- Pass a `name` to emit hidden inputs (one per selected value) for native form
  submission.

## Responsive behaviour

The field is **`width: 100%`** of its container — size it by constraining the
parent. The popover spans the field's full width and scrolls (`max-h`) when the
list is long, so it stays usable on small screens.

```jsx
<div className="max-w-sm">
  <Dropdown label="Country" options={countries} />
</div>
```
