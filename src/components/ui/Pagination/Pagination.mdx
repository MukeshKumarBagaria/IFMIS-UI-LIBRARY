# Pagination

A pill-shaped **page navigator** — prev/next caret steppers around a row of
page numbers. The active page is a Purple-500 (#8B39C2) circle with white
semibold text; the carets disable (and stop responding) at the first / last
page. Long ranges collapse around the current page with ellipses.

Works **controlled** (`page` + `onPageChange`) or **uncontrolled**
(`defaultPage`), mirroring the rest of the library.

> All examples are plain JavaScript / JSX — your modules don't need TypeScript.

---

## Quickstart

```jsx
import { Pagination } from "@ifmis/ui";

// Uncontrolled — the pager tracks its own page; you just react to changes.
<Pagination count={4} defaultPage={1} onPageChange={(page) => load(page)} />
```

## Controlled vs uncontrolled

**Uncontrolled** — pass `defaultPage` and read changes via `onPageChange`. Best
when the pager is the single source of truth.

```jsx
<Pagination count={10} defaultPage={1} onPageChange={fetchPage} />
```

**Controlled** — you own `page` (e.g. it comes from the URL or table state):

```jsx
const [page, setPage] = useState(1);

<Pagination count={10} page={page} onPageChange={setPage} />;
```

## Truncation — long ranges

When the page count is large the range collapses around the current page with
ellipses. Tune how many pages stay visible:

- `siblingCount` — pages shown on **each side** of the current page (default `1`).
- `boundaryCount` — pages always pinned at the **very start / end** (default `1`).

```jsx
{/* count=10, page=5 → 1 … 4 5 6 … 10 */}
<Pagination count={10} defaultPage={5} />

{/* Show two siblings either side → 1 … 3 4 5 6 7 … 10 */}
<Pagination count={20} defaultPage={10} siblingCount={2} />
```

<Canvas of={PaginationStories.Truncated} />

The same algorithm is exported as `paginationRange` if you need the item list
for a custom renderer:

```jsx
import { paginationRange } from "@ifmis/ui";

paginationRange({ count: 10, page: 5, siblingCount: 1, boundaryCount: 1 });
// → [1, "start-ellipsis", 4, 5, 6, "end-ellipsis", 10]
```

## Without the caret steppers

Set `hideControls` to drop the prev/next chips and show only the numbers.

```jsx
<Pagination count={5} defaultPage={2} hideControls />
```

## Disabled

`disabled` greys the whole bar and blocks every button.

```jsx
<Pagination count={10} defaultPage={3} disabled />
```

## In a table footer

Pair `Pagination` with `Dropdown` to build the full “Rows per page” / “Go to
page” toolbar from the design.

```jsx
<div className="flex flex-wrap items-center gap-8">
  <Dropdown label="Rows per page" defaultValue="10" options={rowOptions} />
  <Pagination count={total} page={page} onPageChange={setPage} />
  <Dropdown label="Go to page" value={String(page)} onValueChange={(v) => setPage(Number(v))} options={pageOptions} />
</div>
```

## Props

| Prop            | Type                     | Default        | Notes                                                       |
| --------------- | ------------------------ | -------------- | ----------------------------------------------------------- |
| `count`         | `number`                 | —              | **Required.** Total number of pages.                        |
| `page`          | `number`                 | —              | Controlled current page (1-based). Pair with `onPageChange`.|
| `defaultPage`   | `number`                 | `1`            | Uncontrolled initial page (1-based).                        |
| `onPageChange`  | `(page: number) => void` | —              | Fires with the next page (1-based) on every change.         |
| `siblingCount`  | `number`                 | `1`            | Pages shown on each side of the current page.               |
| `boundaryCount` | `number`                 | `1`            | Pages always shown at the very start / end.                 |
| `hideControls`  | `boolean`                | `false`        | Hide the prev/next caret buttons.                           |
| `disabled`      | `boolean`                | `false`        | Disable the entire pager.                                   |
| `aria-label`    | `string`                 | `"Pagination"` | Accessible name for the `<nav>` landmark.                   |
| `className`     | `string`                 | —              | Merged onto the outer `<nav>`.                              |

Other `<nav>` attributes (`id`, `data-*`, `ref`…) are forwarded. Pages are
internally clamped to `[1, count]`, so out-of-range `page` / `defaultPage`
values are safe.

## Accessibility

- The bar is a `<nav>` landmark named by `aria-label` (default `"Pagination"`),
  so assistive tech can jump straight to it.
- Each number is a real `<button>` labelled `"Go to page N"`; the current page
  carries `aria-current="page"`.
- The caret chips are labelled `"Previous page"` / `"Next page"` and use the
  native `disabled` state at the range ends, so they're skipped by keyboard and
  screen readers.
- Ellipsis markers are `aria-hidden` decoration.
- Every interactive chip shows a visible focus ring (`ring-ring`) on keyboard
  focus.
