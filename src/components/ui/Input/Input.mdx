# Input

A single-line text field. Sized for forms, with optional **start/end
adornments**, three **sizes**, and three **validation states** that drive
both the border colour and the focus ring.

> All examples are plain JavaScript / JSX — your modules don't need TypeScript.

---

## Basics

```jsx
import { Input } from "@ifmis/ui";

<Input placeholder="Search beneficiaries…" />
<Input defaultValue="Hello" />
<Input value={q} onChange={(e) => setQ(e.target.value)} />
```

The component is a thin paint layer over the native `<input>`, so anything
you'd normally pass — `type`, `value`, `onChange`, `name`, `required`,
`autoComplete`, `pattern`, `min`/`max`, … — flows through unchanged.

---

## Sizes

Three sizes; `md` is the default and matches the form spec.

```jsx
<Input size="sm" />   {/* 32px tall */}
<Input size="md" />   {/* 40px tall (default) */}
<Input size="lg" />   {/* 44px tall */}
```

Pair the input size with the surrounding `Button` size — `sm` ↔ `Button
size="small"`, `md` / `lg` ↔ `Button size="standard"`.

---

## Validation states

```jsx
<Input state="default" />
<Input state="error" />    {/* red border + red focus ring */}
<Input state="success" />  {/* green border + green focus ring */}
```

`state` is **purely visual** — pair it with a label/message component
to convey the meaning:

```jsx
<Label htmlFor="email">Email</Label>
<Input
  id="email"
  type="email"
  state={touched && !valid ? "error" : "default"}
  aria-invalid={!valid}
  aria-describedby={!valid ? "email-error" : undefined}
/>
{!valid && (
  <p id="email-error" className="text-body-xs text-red-700">
    Please enter a valid email address.
  </p>
)}
```

## Adornments

A start or end adornment renders **inside** the input, padded around the
text. Use it for icons, units, or short prefixes.

```jsx
import { MagnifyingGlass, X } from "@phosphor-icons/react";

{/* Search box */}
<Input
  placeholder="Search…"
  startAdornment={<MagnifyingGlass />}
/>

{/* Clearable input */}
<Input
  value={q}
  onChange={(e) => setQ(e.target.value)}
  endAdornment={
    q ? (
      <button onClick={() => setQ("")} aria-label="Clear">
        <X />
      </button>
    ) : null
  }
/>

{/* Currency suffix */}
<Input endAdornment={<span className="text-body-secondary">INR</span>} />
```

When an `endAdornment` is a `<button>`, give it an `aria-label` — the
adornment slot doesn't auto-label anything.

---

## Pair with `Label`

The library exports a `Label` primitive that handles styling + `htmlFor`
plumbing. Always link them via `id` / `htmlFor` so clicking the label focuses
the input:

```jsx
import { Label, Input } from "@ifmis/ui";

<div className="flex flex-col gap-1.5">
  <Label htmlFor="grn">GRN number</Label>
  <Input id="grn" placeholder="Enter GRN" />
</div>
```

## Common patterns

### Controlled with debounce (search bar)

```jsx
const [q, setQ] = useState("");
const debounced = useDebounce(q, 250);

useEffect(() => {
  if (debounced) search(debounced);
}, [debounced]);

<Input
  startAdornment={<MagnifyingGlass />}
  placeholder="Search vouchers…"
  value={q}
  onChange={(e) => setQ(e.target.value)}
/>
```

### Numeric input (rupees)

```jsx
<Input
  type="number"
  inputMode="decimal"
  min="0"
  step="0.01"
  startAdornment={<span>₹</span>}
/>
```

### Password with show/hide toggle

```jsx
const [show, setShow] = useState(false);

<Input
  type={show ? "text" : "password"}
  endAdornment={
    <button type="button" onClick={() => setShow((v) => !v)} aria-label={show ? "Hide password" : "Show password"}>
      {show ? <EyeSlash /> : <Eye />}
    </button>
  }
/>
```

## Responsive behaviour

The input is **`width: 100%`** of its container by default. Constrain it by
wrapping in a sized parent:

```jsx
<div className="max-w-md">
  <Input placeholder="GRN number" />
</div>
```

Inputs in a responsive form row:

```jsx
<div className="grid grid-cols-1 gap-4 sm:grid-cols-2">
  <Input placeholder="First name" />
  <Input placeholder="Last name" />
</div>
```

## Props

| Prop              | Type                            | Default   | Notes                                              |
| ----------------- | ------------------------------- | --------- | -------------------------------------------------- |
| `size`            | `sm \| md \| lg`                | `md`      | 32 / 40 / 44 px tall.                              |
| `state`           | `default \| error \| success`   | `default` | Border + focus-ring colour.                        |
| `startAdornment`  | node                            | —         | Inside the input, on the left.                     |
| `endAdornment`    | node                            | —         | Inside the input, on the right.                    |
| `className`       | string                          | —         | Merged onto the input.                             |

All native `<input>` attributes (`type`, `value`, `defaultValue`, `onChange`,
`name`, `placeholder`, `required`, `id`, `ref`, …) are forwarded. **The
component forces `type="text"` by default** — pass `type="email"`,
`"password"`, `"number"`, etc. as needed.

---

## Accessibility

- A real native `<input>` underneath — focus, keyboard and AT support for
  free.
- Always pair with a `<Label htmlFor={id}>` so clicking the label focuses
  the input.
- For validation messages, link the message to the input via
  `aria-describedby` and set `aria-invalid` on error so screen readers
  announce the relationship.
- For a search/combobox role, set `role="searchbox"` or `role="combobox"`
  explicitly.
- The focus ring respects `:focus-visible` — it appears on keyboard focus
  only, not on mouse click.
