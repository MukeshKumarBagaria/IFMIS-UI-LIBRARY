# ReferenceIdSuccessCard

The IFMIS **“Submitted Successfully”** confirmation card — a green-gradient
surface with a success badge, a title, the reference id in a big **dashed
pill**, a supporting line, and a one-click **Copy** button.

Every piece of text is a prop, so the same card covers any _success + reference
id_ moment: a grievance, a payment, a registration receipt, and so on.

> All examples are plain JavaScript / JSX — your modules don't need TypeScript.

---

## Quickstart

Only `referenceId` is required; everything else has a sensible default.

```jsx
import { ReferenceIdSuccessCard } from "@ifmis/ui";

<ReferenceIdSuccessCard
  referenceId="SR - BM-BPL - 0001 - 000234"
  onCopy={(id) => console.log("copied", id)}
/>;
```

## Customising the text

`title`, `description`, `copyLabel`, and `copiedLabel` are all props. Repoint
them and the card becomes a payment receipt, a registration confirmation, or
anything else.

```jsx
<ReferenceIdSuccessCard
  title="Payment Successful!"
  referenceId="TXN-2026-000912"
  description="Your payment has been recorded in the IFMIS-Next Gen System"
  copyLabel="Copy Transaction ID"
/>
```

## Copy to clipboard

Clicking **Copy** writes `copyValue` to the clipboard, flips the button to its
`copiedLabel` for `copiedDuration` ms (default `2000`), then fires `onCopy`.

- `copyValue` defaults to `referenceId` when it's a string. If you pass a node
  for `referenceId`, set `copyValue` explicitly so the right text is copied.
- If the Clipboard API rejects (insecure context / denied permission), the card
  still flips to the copied state and calls `onCopy`, so you can provide a
  fallback (e.g. a toast).

```jsx
<ReferenceIdSuccessCard
  referenceId={<strong>SR - 0001</strong>}
  copyValue="SR-0001"
  copiedLabel="Copied!"
  onCopy={(id) => toast(`Copied ${id}`)}
/>
```

## Props

| Prop                  | Type                       | Default                              | Notes                                                  |
| --------------------- | -------------------------- | ------------------------------------ | ------------------------------------------------------ |
| `referenceId`         | node                       | — (required)                         | The id shown large in the dashed pill.                 |
| `title`               | node                       | `"Grievance Submitted Successfully!"`| Success message title.                                 |
| `description`         | node                       | IFMIS registration line              | Supporting line under the id.                          |
| `icon`                | node \| `null`             | green check badge                    | `null` hides it; a node replaces it.                   |
| `showCopyButton`      | boolean                    | `true`                               | Toggle the copy button.                                |
| `copyLabel`           | node                       | `"Copy Grievance ID"`                | Resting copy-button label.                             |
| `copiedLabel`         | node                       | `"Copied!"`                          | Label while in the copied state.                       |
| `copyValue`           | string                     | `referenceId` (if string)            | Exact text written to the clipboard.                   |
| `copiedDuration`      | number                     | `2000`                               | How long (ms) the copied state lasts.                  |
| `onCopy`              | `(value: string) => void`  | —                                    | Fires after a copy.                                    |
| `className`           | string                     | —                                    | Classes for the card root.                             |
| `idClassName`         | string                     | —                                    | Classes for the dashed id pill.                        |
| `copyButtonClassName` | string                     | —                                    | Classes for the copy button.                           |

Also forwards a `ref` and any extra `<div>` attributes to the root.

## Accessibility

- The copy control is a native `<button>` — keyboard-focusable, activates with
  <kbd>Enter</kbd> / <kbd>Space</kbd>, and shows a visible focus ring
  (`ring-green-800`).
- It carries `aria-live="polite"`, so when the label flips to "Copied!" after a
  successful copy, screen readers announce the confirmation.
- The success badge and the copy/check icons are `aria-hidden` decoration; the
  meaning lives in the visible text.
- `navigator.clipboard` may be unavailable (insecure origin / older browser) —
  `onCopy` still fires so you can surface a manual-copy fallback.
- Colour is decorative; the title and id text carry the success message
  independently of the green theme.

## Responsive behaviour

The card is **`width: 100%`** up to a `657px` max (the Figma reference width) —
size it by constraining the parent. The id wraps inside its pill, so long
references stay readable on small screens.

```jsx
<div className="max-w-[657px]">
  <ReferenceIdSuccessCard referenceId="SR - BM-BPL - 0001 - 000234" />
</div>
```
