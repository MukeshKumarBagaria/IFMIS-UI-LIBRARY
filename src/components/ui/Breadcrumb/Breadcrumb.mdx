# Breadcrumb

The page-trail navigation chips that show where the user is. The **last**
crumb is the current page (purple text on a neutral-100 chip); the rest are
muted grey chips, separated by a caret. It takes **any number of items** and
infers whether each crumb is a link, a button, or plain text.

> All examples are plain JavaScript / JSX — your modules don't need
> TypeScript.

---

## Quickstart

Pass an `items` array, root → current. The last item is the active page.

```jsx
import { Breadcrumb } from "@ifmis/ui";

<Breadcrumb
  items={[
    { label: "Home", href: "/" },
    { label: "Vouchers", href: "/vouchers" },
    { label: "VCH-2026-0042" }, // no href/onClick → current page
  ]}
/>
```

## How each crumb renders

The element is inferred from the item's props — you don't choose a type:

| Item has…            | Renders as | Notes                              |
| -------------------- | ---------- | ---------------------------------- |
| `href`               | `<a>`      | Standard anchor navigation         |
| `onClick` (no href)  | `<button>` | Great for SPA routers              |
| neither              | `<span>`   | Plain text                         |
| is the current crumb | `<span>`   | Always non-interactive, `aria-current="page"` |

```jsx
// Router buttons
<Breadcrumb
  items={[
    { label: "Dashboard", onClick: () => navigate("/") },
    { label: "HRMS", onClick: () => navigate("/hrms") },
    { label: "Leave" },
  ]}
/>
```

## Customise

### Leading icon

Add an `icon` to any item (auto-sized to 16px):

```jsx
import { House } from "@phosphor-icons/react";

<Breadcrumb
  items={[
    { label: "Home", href: "/", icon: <House weight="fill" /> },
    { label: "Reports", href: "/reports" },
    { label: "Monthly" },
  ]}
/>
```

<Canvas of={BreadcrumbStories.WithIcon} />

### Separator

Pass any node — an icon, a slash, a dot:

```jsx
<Breadcrumb separator={<span className="text-grey-500">/</span>} items={...} />
```

<Canvas of={BreadcrumbStories.CustomSeparator} />

### Which crumb is active

By default it's the last item. Override by setting `current: true` on any item
(useful when the trail continues past the current page):

```jsx
<Breadcrumb
  items={[
    { label: "A", href: "/a" },
    { label: "B", current: true },
    { label: "C", href: "/c" },
  ]}
/>
```

### Other props

| Prop        | Type                | Default        | Notes                                   |
| ----------- | ------------------- | -------------- | --------------------------------------- |
| `items`     | `BreadcrumbItem[]`  | —              | The trail, in order.                    |
| `separator` | node                | caret-right    | Rendered between crumbs.                |
| `ariaLabel` | string              | `"Breadcrumb"` | Label for the `<nav>` landmark.         |
| `className` | string              | —              | Applied to the `<nav>`.                 |

Each `BreadcrumbItem`: `{ label, href?, onClick?, icon?, current?, className?, key? }`.

## Tokens

| Part                | Background           | Text                  |
| ------------------- | -------------------- | --------------------- |
| Current crumb       | `#F1EDF5` (neutral-100) | `#470D73` (purple-800) |
| Other crumbs        | `#F7F7F7` (grey-bg)  | `#595959` (body-secondary) |

Chip: `padding 4px 8px`, `border-radius 16px`, `gap 10px`, text 14px / 600.

## Accessibility

- Wrapped in `<nav aria-label="Breadcrumb">` with an ordered list (`<ol>`).
- The current crumb carries `aria-current="page"`.
- Separators are `aria-hidden` so screen readers don't announce them.
- Button crumbs get a visible focus ring.

✅ Keep the current page as the last item (or mark it `current`).
✅ Use `href` for real navigation so links are crawlable and openable in a new tab.
❌ Don't make the current crumb a link — it's the page you're already on.

## Responsive behaviour

Breadcrumb chips **wrap onto multiple lines** on narrow screens (they sit in a
flex row with a gap). For deep trails on mobile, collapse the middle crumbs and
keep the first + last:

```jsx
{/* Show all on desktop; first + current on mobile */}
const trail = isMobile
  ? [items[0], { label: "…", current: false }, items[items.length - 1]]
  : items;

<Breadcrumb items={trail} />
```

Keep labels short — long crumb labels are the main cause of awkward wrapping.
