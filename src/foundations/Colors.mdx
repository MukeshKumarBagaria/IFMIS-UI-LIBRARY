# Colors

Every colour in `@ifmis/ui` comes from a token in one of three layers:

1. **Brand palettes** — raw ramps shared across themes (green, red,
   orange, yellow, blue, grey).
2. **Theme palettes** — primary (`purple`) and tinted neutral. Different per
   theme.
3. **Semantic tokens** — the theme-agnostic API (`--primary`,
   `--background`, `--success`, surface, text). This is what components
   consume.

> Every swatch on this page is rendered by reading the **live CSS variable**
> at render time. If a colour ever drifts from Figma, this page is the first
> thing that flags it. Flip the Storybook theme toolbar to see the
> theme-dependent tokens update.

---

## How to consume a colour

In components, never write a hex. Use the Tailwind utility named after the
token. `globals.css` ties every `--brand-*` / `--color-*` variable to a
class:

```jsx
// good
<div className="bg-green-100 border border-green-400 text-green-800">…</div>
<div className="bg-primary text-primary-foreground">…</div>

// bad — hardcoded, doesn't theme
<div style={{ background: "#0F8A49" }}>…</div>
```

When the same colour appears in a *theme-dependent* place (page chrome,
brand button, focus ring), prefer the semantic token (`bg-primary`) so it
follows the theme. Brand ramps (`bg-green-200`) are for *semantic* status
colours that mean the same thing in every theme.

## Best practices

- **Never inline a hex code.** Use the named token / utility every time.
  The first time you "just need this one custom colour" is the moment your
  theme breaks.
- **Pair semantic colour with semantic markup.** A red surface (`bg-red-100`)
  needs `role="alert"`-grade copy on it — don't use the danger palette for
  pure decoration.
- **Prefer semantic tokens over raw palette steps** when the meaning is
  theme-shared. `bg-primary` over `bg-purple-600` for a brand button.
- **Use the 25 / 50 steps for ambient washes**, 100–300 for chips and
  surfaces, 400–600 for icons and borders, 700–900 for text.
- **Status colour is reinforcement, not signal.** Always pair status colour
  with a label or icon — never carry meaning with colour alone (WCAG SC
  1.4.1).

## Versioning

The token names on this page are the **public contract**. Adding a new
token (`--brand-teal-500`) is a minor bump; renaming or removing one is
a major. See PUBLISHING.md.
